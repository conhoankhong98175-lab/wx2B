# 简要威胁模型

| 风险                      | 控制                                                                             |
| ------------------------- | -------------------------------------------------------------------------------- |
| 猜测报价编号访问客户价格  | 客户访问只接受带 HMAC 签名和 256 位随机 nonce 的私密 token                       |
| 数据库或备份泄漏          | 敏感字段加密；备份使用独立密钥 AES-256-GCM 加密；限制文件 ACL                    |
| 成本通过接口或导出泄漏    | 成本权限实时校验；客户 DTO 白名单；PDF/长图只消费公开文档模型                    |
| A 商家访问 B 商家数据     | 所有商家查询带 merchant_id；分类和附加项适用产品写入时校验租户                   |
| 价格库变化篡改历史        | 正式版本保存加密快照、内容哈希和规则版本；数据库触发器禁止更新                   |
| 普通代码删除正式证据      | 数据库触发器禁止删除正式版本；只有已确认的账户删除事务临时授权级联删除           |
| 弱网重复发布或重复接受    | 客户动作使用请求标识与请求摘要；版本号有唯一约束；发布使用草稿 revision CAS      |
| 撤回与客户接受竞争        | 客户动作在事务内重新检查状态和有效期；撤回只允许 ACTIVE/CHANGE_REQUESTED         |
| 公开 PDF 消耗 CPU         | 验签后按版本限流；无效随机 token 不进入限流 Map                                  |
| AppSecret 被小程序反编译  | 小程序只传 wx.login code；AppSecret 只存在服务器环境变量                         |
| Electron 渲染进程接管本机 | nodeIntegration=false、contextIsolation=true、sandbox=true、拒绝权限/导航/新窗口 |

已接受的边界：SQLite 仅用于单写实例；横向扩容前必须迁移支持多写并发的数据库。Windows 未代码签名的便携包可能出现 SmartScreen 提示，正式广泛分发时需配置 Authenticode 证书。
