# 微信小程序配置与发布

## 需要准备

正式发布前由小程序管理员准备：

1. 已完成必要备案的小程序账号和 AppID。
2. 小程序主体、名称、图标、简介、服务类目及相应资质。
3. 已备案且证书有效的 HTTPS API 域名。
4. 隐私联系邮箱、数据保存期限和注销规则。
5. AppSecret。请直接写入服务器 Secret，不要发送到聊天或提交 Git。

## 导入工程

1. 用微信开发者工具导入仓库中的 `miniprogram/`。
2. 在私有项目配置中设置真实 AppID。
3. 修改 `miniprogram/config.js` 的 `apiBaseUrl`。
4. 开发阶段可临时关闭域名校验；真机和提审前必须重新启用。

## 服务器域名

在“小程序后台 → 开发 → 开发设置 → 服务器域名”配置：

- request 合法域名：API HTTPS 域名。
- downloadFile 合法域名：PDF 下载使用的同一域名。
- 若使用外部 Logo，其 HTTPS 主机也必须满足微信图片资源域名要求；推荐与 API 使用同一已备案主机。

不能使用 IP、localhost 或无有效证书的域名；不要把 `api.weixin.qq.com` 配成小程序业务域名。

## 隐私接口

本项目可能使用：

- `wx.login`：登录并绑定商家账号。
- `wx.saveImageToPhotosAlbum`：由用户主动保存报价长图。
- `wx.shareFileMessage`：由用户主动转发数据导出文件。

未使用手机号、定位、通讯录、麦克风或广告 SDK。提审时隐私保护指引必须与实际接口一致，并提供联系邮箱、保存期限、更正与删除方式。

## 真机验收

- iOS 与 Android 各完成一次：登录 → 价格库 → 草稿 → 发布 → 客户页 → 修改 → V2 → 接受。
- PDF 能下载、中文无缺字、`openDocument(showMenu: true)` 可保存或转发。
- 长图保存权限拒绝时有明确提示，再次授权后可成功。
- 旧版本、过期、接受、撤回和被替代状态均正确。
- 分享卡不含客户电话、总价和其他敏感信息；只有私密 token。
- 开发者工具重新启用域名和 TLS 校验后无报错。

## 上传发布

开发者工具上传版本后，在后台提交审核并填写服务类目、功能说明和测试路径。若使用 `miniprogram-ci`，代码上传私钥只能保存在 CI Secret 中，并设置 IP 白名单；仓库已忽略相关私有配置。
