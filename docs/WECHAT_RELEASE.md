# 店告微信小程序上架操作手册

> 核验日期：2026-07-16  
> 适用工程：`miniprogram/` 原生微信小程序  
> 推荐主类目：工具 → 报价/比价

本手册把上架工作拆成账号、备案、基础信息、服务器、隐私、代码上传、审核和发布八个关卡。前一关未完成时不要跳到后一关，否则最常见的结果是上传成功但无法提审，或审核人员无法登录。

## 1. 先确定运营主体

在 [微信公众平台官网](https://mp.weixin.qq.com/) 注册“小程序”账号。注册邮箱不能已经用于公众平台、开放平台或企业微信账号。按真实运营者选择个人、个体工商户或企业主体，不要借用无关主体。

本产品面向门店经营，若实际运营者持有个体工商户或企业营业执照，优先使用对应非个人主体；这能让账号名称、品牌权利、备案主体和服务器备案主体保持一致。个人主体是否可用应以注册后台当时开放的类目为准。

官方注册步骤和主体能力说明见[小程序产品定位及功能介绍](https://developers.weixin.qq.com/miniprogram/introduction/)。企业账号可按后台提供的验证方式完成主体验证；政府、媒体和其他组织有强制认证要求。认证材料和时限见[微信认证指引](https://developers.weixin.qq.com/miniprogram/product/renzheng.html)。认证费用和规则可能调整，以公众平台订单页为准。

完成后记录：

- 小程序 AppID：可公开给客户端，不是密钥。
- 小程序管理员微信号：用于扫码、备案核身、提审和发布。
- AppSecret：只写入服务器 Secret，不能写进小程序、仓库、文档或聊天。
- 若另行启用自动化 CI，代码上传密钥与 AppSecret 不是同一个东西；两者都不能进入仓库或聊天。本项目默认使用已登录的微信开发者工具 CLI，不需要代码上传密钥。

## 2. 完善名称、图标、简介和服务类目

建议提交内容已整理在 [WECHAT_SUBMISSION_COPY.md](WECHAT_SUBMISSION_COPY.md)。最终名称需要在公众平台检查是否可用，并确保运营主体拥有相应名称或商标权利。

推荐：

- 名称：`店告报价助手`；若被占用可用“主体简称 + 店告报价”。
- 主类目：`工具 → 报价/比价`。
- 可选补充类目：`工具 → 办公` 或 `商业服务 → 企业管理`，仅在后台允许且确实需要时添加。
- 不建议选 `广告/设计`：产品服务是报价与门店办公工具，不是向用户提供广告设计服务。

微信官方类目表对“报价/比价”的说明是商品报价或价格对比，且当前表格未列额外资质，和本产品最吻合。类目会随主体类型和政策变化，提交前仍应在[小程序开放的服务类目](https://developers.weixin.qq.com/miniprogram/product/material/)复核。

基本信息注意事项：

- 名称、图标、简介必须互相关联，并准确表达真实功能。
- 不使用“微信官方”“国家级”“最佳”“免费清仓”等易误导或商业化极限词。
- 图标必须清晰，有色背景或边框，不使用微信、腾讯或无授权第三方标识。
- 首页应在两次点击内到达所申报类目的核心页面。本项目首页可直接创建报价，并可一键进入价格库，符合这一结构。

详见[平台运营规范](https://developers.weixin.qq.com/miniprogram/product/)和[常见拒绝情形](https://developers.weixin.qq.com/miniprogram/product/reject.html)。

## 3. 完成小程序备案

未上架的新小程序需要先备案。入口是公众平台首页的“去备案”，或“设置 → 小程序备案”。官方流程分五段：

1. 填写备案信息。
2. 微信平台初审，官方指引给出的参考时间为 1–2 个工作日。
3. 工信部短信核验。短信来自 12381，主体负责人和小程序负责人收到后必须在 24 小时内前往 [工信部备案系统](https://beian.miit.gov.cn/)核验；超时会被自动驳回。
4. 省通信管理局审核，官方参考时间为 1–20 个工作日。
5. 下发小程序备案号。发布后备案号会自动展示在小程序更多资料页。

通常需要提前准备：

- 主办单位证件。
- 主体负责人证件和本人手机号。
- 小程序负责人证件和本人手机号。
- 真实通讯地址。
- 负责人现场拍照及人脸核验。
- 授权书、居住证或地方管局要求的补充材料。
- 若涉及新闻、出版、药品、医疗、网约车等特殊业务，还需对应前置审批材料。本项目本身是办公报价工具，正常情况下不涉及这些前置审批；不要在简介或页面中加入未获许可的行业功能。

完整材料和地方差异见[小程序备案操作指引](https://developers.weixin.qq.com/miniprogram/product/record_guidelines.html)。

## 4. 准备生产服务器和域名

线上小程序不能连接 Windows 本地版数据库。必须先部署本项目的 `server/`，并准备：

- 已完成 ICP 备案的真实域名，例如 `https://quote.example.com`。
- 有效、受系统信任且证书链完整的 HTTPS 证书。
- TLS 1.2 或以上。
- 一个持久化 SQLite 数据卷和独立备份卷。
- 与该小程序匹配的 `WECHAT_APP_ID`、`WECHAT_APP_SECRET`。
- 至少 32 字节随机 `APP_SECRET` 和独立 `BACKUP_SECRET`。

部署方法见 [DEPLOYMENT.md](DEPLOYMENT.md)。部署后先检查：

```powershell
curl.exe https://你的域名/api/health
```

响应必须包含 `"ok":true`、`"mode":"server"`。

然后登录公众平台：“开发 → 开发管理 → 开发设置 → 服务器域名”，把同一个 HTTPS Origin 配置到：

- request 合法域名。
- downloadFile 合法域名。

本项目没有上传文件和 WebSocket，因此当前不需要 uploadFile、socket 域名。若以后把 Logo 放在另一个域名，还要按微信当前资源域名规则配置；最稳妥的做法是把 Logo 放在 API 同域。

官方限制包括：不能使用 IP 或 localhost；不能把 `api.weixin.qq.com` 配为业务域名；使用子域名时必须精确配置；证书域名、有效期和完整信任链都要正确。详见[小程序网络文档](https://developers.weixin.qq.com/miniprogram/dev/framework/ability/network.html)。

## 5. 配置隐私保护指引

入口：“设置 → 服务内容声明 → 用户隐私保护指引”。每次提审时，平台还会把开发版实际接口与提审版隐私指引比对；只改现网指引不一定能解决开发版提审拦截。

本项目必须如实声明：

1. `使用你的相册（仅写入）权限`：对应 `wx.saveImageToPhotosAlbum`。用途写“用户主动点击保存长图时，将当前报价长图写入系统相册；不读取、扫描或上传相册内容”。
2. 微信登录产生的 OpenID：用于识别商家账号和维持登录态；AppSecret 只在服务器换取登录结果。
3. 商家主动填写的店铺名称、Logo、联系人、电话和微信号：用于报价展示和联系。
4. 商家主动填写的客户名称、可选联系方式、项目、尺寸、数量、价格、备注和条款：用于计算、保存和展示报价。
5. 客户主动提交的接受、问题和修改说明：用于商家跟进。
6. 必要的请求时间、匿名标识和审计摘要：用于防重复提交、安全和排障。

不要声明本项目没有使用的手机号快捷验证、定位、通讯录、麦克风、相册读取或广告 SDK。

平台会根据代码中的隐私接口生成必填项；未声明的隐私接口会被禁用。当前代码已显式启用 `__usePrivacyCheck__`，设置页也提供 `wx.openPrivacyContract` 官方协议入口。微信在开发者未自定义处理时可显示官方隐私弹窗，但仍需在后台先完成真实、审核通过的隐私指引。

需要由运营者确定并填写：

- 运营主体全称。
- 隐私联系邮箱。
- 数据保存期限。
- 服务器所在国家或地区。
- 查询、更正、导出、删除和投诉渠道。

本项目内已有“设置 → 导出数据”和“设置 → 删除店铺”；删除会使公开报价链接失效。技术底稿见 [PRIVACY.md](PRIVACY.md)。官方填写和接口映射见[隐私保护指引填写说明](https://developers.weixin.qq.com/miniprogram/dev/framework/user-privacy/)与[内容介绍](https://developers.weixin.qq.com/miniprogram/dev/framework/user-privacy/miniprogram-intro.html)。

## 6. 把真实 AppID 和域名写入本地工程

不要直接编辑并提交 AppID 私有配置。运行：

```powershell
npm run wechat:configure -- `
  --appid wx1234567890123456 `
  --api-base-url https://你的域名
```

该命令会：

- 把 AppID 写入已被 Git 忽略的 `miniprogram/project.private.config.json`。
- 把非敏感的生产 API 地址写入 `miniprogram/config.js`。

然后执行严格预检：

```powershell
npm run wechat:preflight -- --strict-online
```

报告写入 `out/wechat-preflight.json`。任何 `BLOCK` 都必须在上传前解决；`manualChecks` 需要管理员在公众平台确认。

## 7. 使用开发者工具预览和真机验收

从[微信开发者工具官方下载页](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)安装稳定版，用管理员或已加入项目成员的微信扫码登录，导入仓库的 `miniprogram/` 目录。

在“详情 → 本地设置”确认：

- AppID 正确。
- 基础库使用稳定版本。
- 不勾选“不校验合法域名、web-view、TLS 版本以及 HTTPS 证书”。
- `urlCheck` 保持开启。

开发者工具模拟器不能代替真机。至少在 iOS 和 Android 各跑一次：

1. 首次进入，`wx.login` 自动创建商家账号。
2. 修改店铺信息和价格库。
3. 创建、预览并发布 V1 报价。
4. 客户页查看、提问、申请修改和接受。
5. 创建 V2，旧链接显示已被替代。
6. PDF 可打开且中文无缺字。
7. 长图首次保存、拒绝权限、打开设置恢复、分页保存均正确。
8. 分享卡不包含电话、客户联系方式、金额等敏感信息。
9. 导出数据和永久删除账号可用。
10. 隐私说明按钮能打开公众平台配置的协议。

## 8. 上传开发版本

### 方法 A：开发者工具

点击右上角“上传”，填写版本号和项目备注。上传后登录公众平台，在“管理 → 版本管理”看到开发版本。

### 方法 B：本项目调用开发者工具 CLI

安装并登录微信开发者工具，在“设置 → 安全设置”开启服务端口。若工具不在默认路径，用 `WECHAT_DEVTOOLS_CLI` 指向安装目录中的 `cli.bat`。

```powershell
npm run wechat:login
npm run wechat:preview
npm run wechat:upload -- --version 1.0.0 --desc '首次提审：报价、版本和客户确认闭环'
```

登录和预览二维码分别输出到 `out/wechat-login.png`、`out/wechat-preview.png`。上传命令会先执行生产 API 在线预检。命令行参数和登录要求见[微信开发者工具 CLI 文档](https://developers.weixin.qq.com/miniprogram/dev/devtools/cli.html)。

## 9. 设置体验版

公众平台“版本管理”中，把刚上传的开发版本选为体验版；在“成员管理”添加真实测试人员为体验成员。让非开发人员按上一节的十步验收一次，重点观察授权弹窗、域名、真实网络和分享行为。

## 10. 提交审核

管理员登录公众平台：“管理 → 版本管理 → 开发版本 → 提交审核”。

填写：

- 服务类目：工具 → 报价/比价。
- 重要业务页面：首页、创建报价、价格库、客户报价页；不超过后台允许的页面组数。
- 功能说明和测试路径：直接复制 [WECHAT_SUBMISSION_COPY.md](WECHAT_SUBMISSION_COPY.md)。
- 测试账号：本项目不需要账号密码，审核员用微信进入后会通过 `wx.login` 创建独立商家空间。说明这一点，避免审核员寻找账号。
- 隐私保护指引：确认提审版已声明相册写入和所有实际处理的信息。

提交后在审核版本查看进度。审核通过不等于上线，管理员还必须手动点击“发布”。官方完整流程见[小程序注册、审核与发布介绍](https://developers.weixin.qq.com/miniprogram/introduction/#%E4%B8%89%E3%80%81%E4%BB%A3%E7%A0%81%E5%AE%A1%E6%A0%B8%E4%B8%8E%E5%8F%91%E5%B8%83)。

## 11. 首次发布和发布后检查

审核通过后：

1. 管理员点击“发布”。
2. 用不在开发/体验成员列表中的普通微信搜索小程序并打开。
3. 重新完成登录、报价、客户页、PDF、长图和删除账号冒烟。
4. 检查“更多资料”中的备案号、隐私保护指引、主体和服务类目。
5. 监控服务器 `/api/health`、5xx、磁盘、SQLite WAL 和备份任务。
6. 立即执行一次加密备份并在隔离环境恢复。

版本更新继续使用“上传 → 体验 → 提审 → 发布”，不要直接跳过体验版。隐私接口或数据用途变化时，先更新提审版隐私指引。

## 12. 当前仍需要运营者提供的内容

代码无法替代以下事项：

- 已注册账号的真实 AppID。
- 主体类型及备案状态。
- 已备案的生产 HTTPS 域名。
- 运营主体全称、隐私邮箱、数据期限和服务器地区。
- 管理员扫码、人脸核身、12381 短信核验、提审和最终发布点击。
- AppSecret 在服务器的安全配置；若自行使用 CI，再单独保管代码上传密钥。

不要在聊天中发送 AppSecret、代码上传密钥、身份证照片、营业执照原图或短信验证码。
